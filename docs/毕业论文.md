# 基于Hive的城市空气质量分析与预测系统的设计与实现

## 毕业论文

---

## 摘要

随着我国城市化进程加快，空气质量问题日益成为社会关注的焦点。本文设计并实现了一个基于Hive的城市空气质量分析与预测系统，采用Hadoop/Hive作为数据仓库核心，结合Spark进行分布式数据处理与分析，系统涵盖数据采集、ETL清洗、多维分析、预测建模和可视化展示等功能模块。

系统采用ODS→DWD→DWS→ADS分层架构设计数据仓库，存储超过12个月的郑州市小时级空气质量监测数据和气象数据。在多维分析方面，实现了时空分布分析、污染物相关性分析、时间序列趋势分析。在预测建模方面，集成XGBoost、随机森林、LSTM等算法，构建了24-72小时的AQI预测模型，AQI等级预测准确率达到80%以上。

系统采用Spring Boot + Vue.js技术栈开发Web可视化平台，实现了实时AQI看板、污染分布热力图、趋势分析图、预测结果展示等功能，为环境管理部门提供决策支持。

**关键词**：空气质量分析；Hive数据仓库；Spark；预测模型；数据可视化

---

## Abstract

With the acceleration of urbanization in China, air quality has become a growing public concern. This paper designs and implements an air quality analysis and prediction system based on Hive. Using Hadoop/Hive as the core of data warehouse and Spark for distributed data processing and analysis, the system covers data collection, ETL cleaning, multi-dimensional analysis, prediction modeling and visualization.

The system adopts ODS→DWD→DWS→ADS layered architecture to design the data warehouse, storing over 12 months of hourly air quality monitoring data and meteorological data from Zhengzhou city. In terms of multi-dimensional analysis, it implements spatial-temporal distribution analysis, pollutant correlation analysis, and time series trend analysis. For prediction modeling, it integrates XGBoost, Random Forest, LSTM and other algorithms to build 24-72 hour AQI prediction models, with AQI level prediction accuracy reaching over 80%.

The system uses Spring Boot + Vue.js technology stack to develop the Web visualization platform, implementing real-time AQI dashboard, pollution distribution heat map, trend analysis charts, prediction results display and other functions, providing decision support for environmental management departments.

**Keywords**: Air Quality Analysis; Hive Data Warehouse; Spark; Prediction Model; Data Visualization

## 第二章 相关技术与方法

### 2.1 Hadoop生态系统

#### 2.1.1 HDFS分布式文件系统

HDFS（Hadoop Distributed File System）是Hadoop的分布式文件系统，采用主从架构设计。一个HDFS集群由一个NameNode和多个DataNode组成。NameNode负责管理文件系统的元数据，包括文件目录结构、文件块位置等信息。DataNode负责存储实际的数据块，每个数据块默认有3个副本，分布在不同的DataNode上，确保数据的可靠性和容错性。

HDFS具有高容错性、高吞吐量、适合处理大文件等特点，但不适合低延迟的数据访问场景。

#### 2.1.2 MapReduce编程模型

MapReduce是Google提出的分布式计算编程模型，将计算过程分为Map和Reduce两个阶段。Map阶段将输入数据映射为键值对，Reduce阶段对相同键的值进行合并处理。

MapReduce适合处理批处理任务，但编程模型较为复杂，且中间结果需要写入磁盘，效率较低。

#### 2.1.3 YARN资源调度

YARN（Yet Another Resource Negotiator）是Hadoop 2.0引入的资源管理系统，将资源管理和任务调度分离。YARN包含ResourceManager、NodeManager、ApplicationMaster三个组件。ResourceManager负责整个集群的资源管理，NodeManager负责单个节点的资源管理，ApplicationMaster负责单个应用程序的资源申请和任务协调。

### 2.2 Hive数据仓库

#### 2.2.1 Hive架构

Hive是建立在Hadoop之上的数据仓库工具，提供了类似SQL的查询语言（HiveQL）来操作存储在HDFS中的数据。Hive的架构包含：

1. **用户接口**：CLI、JDBC/ODBC、Web UI
2. **Driver**：接收查询请求，管理会话
3. **Compiler**：解析查询语句，生成执行计划
4. **Optimizer**：优化执行计划
5. **Execution Engine**：执行任务
6. **Metastore**：存储元数据
7. **Storage Handler**：与底层存储交互

#### 2.2.2 分区与桶

Hive支持分区（Partition）和桶（Bucket）两种数据组织方式。分区按照某个或多个列的值将数据划分到不同的目录，适合按分区列进行查询的场景。桶将数据按照某个列的哈希值划分到不同的文件，可以优化JOIN操作和采样查询。

#### 2.2.3 数据类型与存储格式

Hive支持丰富的数据类型，包括基本类型（INT、STRING、DOUBLE等）和复杂类型（ARRAY、MAP、STRUCT）。存储格式支持TEXTFILE、SEQUENCEFILE、ORC、Parquet等，其中ORC（Optimized Row Columnar）是列式存储格式，具有较高的压缩比和查询性能。

### 2.3 Spark计算框架

#### 2.3.1 Spark架构

Spark是一个通用的大数据处理引擎，采用内存计算模式，计算速度比MapReduce快10-100倍。Spark架构包含：

- **Spark Core**：提供基础功能，包括任务调度、内存管理、错误恢复
- **Spark SQL**：结构化数据处理，支持DataFrame和Dataset API
- **Spark Streaming**：流式数据处理
- **MLlib**：机器学习库
- **GraphX**：图计算

#### 2.3.2 RDD与DataFrame

RDD（Resilient Distributed Dataset）是Spark的基本数据结构，是一个不可变的分布式对象集合。RDD支持两种操作：转换操作（如map、filter）返回新的RDD，动作操作（如count、collect）返回结果。

DataFrame是比RDD更高层的抽象，以列的形式组织数据，类似于关系型数据库的表。DataFrame支持SQL操作，且性能优于RDD。

#### 2.3.3 Spark SQL

Spark SQL是Spark的一个模块，用于处理结构化数据。Spark SQL支持：

- 将SQL查询与Spark程序无缝集成
- 从多种数据源读取数据（JSON、Parquet、Hive等）
- 使用DataFrame/Dataset API进行编程
- 提供Hive集成，支持HiveQL查询

### 2.4 时间序列预测方法

#### 2.4.1 传统统计方法

1. **ARIMA模型**
   ARIMA（Auto-Regressive Integrated Moving Average）是经典的时间序列预测模型，由自回归项（AR）、差分项（I）和移动平均项（MA）组成。ARIMA模型可以表示为ARIMA(p,d,q)，其中p是自回归阶数，d是差分阶数，q是移动平均阶数。

2. **SARIMA模型**
   SARIMA（Seasonal ARIMA）在ARIMA基础上加入了季节性分量，适合具有周期性特征的时间序列数据。SARIMA模型可以表示为SARIMA(p,d,q)(P,D,Q)s，其中s是季节周期。

#### 2.4.2 机器学习方法

1. **随机森林**
   随机森林是一种集成学习方法，通过构建多棵决策树并综合其结果来进行预测。随机森林具有抗过拟合能力强、处理高维数据等优点。

2. **XGBoost**
   XGBoost是一种梯度提升决策树算法，通过迭代地添加弱学习器来最小化损失函数。XGBoost支持正则化、缺失值处理、并行计算，在许多机器学习竞赛中取得了优异成绩。

#### 2.4.3 深度学习方法

1. **LSTM网络**
   LSTM（Long Short-Term Memory）是一种特殊的循环神经网络，通过门控机制解决长期依赖问题。LSTM特别适合处理时间序列数据，能够学习数据中的长期模式。

2. **CNN-LSTM模型**
   CNN-LSTM结合了卷积神经网络（CNN）和长短期记忆网络（LSTM），CNN用于提取空间特征，LSTM用于处理时序依赖关系。这种模型在视频分析、序列预测等任务中表现出色。

### 2.5 数据可视化技术

#### 2.5.1 ECharts

ECharts是百度开源的可视化图表库，提供了折线图、柱状图、散点图、饼图、热力图、地图等丰富的图表类型。ECharts支持大数据量渲染、交互式操作、动画效果，适用于数据监控和数据分析场景。

#### 2.5.2 Leaflet

Leaflet是一个轻量级的开源JavaScript地图库，支持多种地图瓦片服务，提供交互式地图展示功能。Leaflet适合开发位置服务类应用，如污染分布热力图、监测站点分布图等。

### 2.6 本章小结

本章介绍了论文涉及的关键技术与方法。Hadoop生态系统提供了分布式存储和计算的基础设施，Hive数据仓库支持海量数据的SQL查询分析，Spark计算框架实现了高效的数据处理和机器学习。时间序列预测方法从传统统计模型到深度学习模型各有特点，需要根据实际场景选择合适的方法。数据可视化技术将分析结果以直观的形式展示给用户。

### 3.3 数据仓库设计

#### 3.3.1 分层架构

采用典型的四层数据仓库架构：

1. **ODS层（Operation Data Store）**
   - 存储原始数据，保持数据原貌
   - 按日期分区，便于数据追溯
   - TEXTFILE格式存储

2. **DWD层（Data Warehouse Detail）**
   - 清洗后的明细数据
   - 维度退化，规范化数据
   - ORC格式存储，Snappy压缩

3. **DWS层（Data Warehouse Summary）**
   - 按主题轻度汇总
   - 宽表设计，支持快速查询
   - 按月分区

4. **ADS层（Application Data Store）**
   - 应用层数据，预计算结果
   - 支持前端直接查询
   - ORC格式存储

#### 3.3.2 表结构设计

详见第二章中的数据库设计部分，主要包括：

- **事实表**：ods_air_quality_raw、dwd_air_quality_dt、dws_station_hour等
- **维度表**：dim_station、dim_time、dim_aqi_level等
- **结果表**：ads_realtime_aqi、ads_aqi_prediction等

#### 3.3.3 分区策略

| 层级 | 分区字段 | 分区粒度 | 存储格式 |
|------|----------|----------|----------|

| DWD | year/month/day | 每日 | ORC |
| DWS | year/month | 每月 | ORC |

#### 4.2.2 数据清洗策略

采用以下清洗策略：

1. **空值处理**
   - 使用前向填充或线性插值填充缺失值
   - 缺失超过50%的站点标记为不可用

2. **异常值检测**
   - 使用3σ原则：超过均值3个标准差的数据视为异常
   - 使用箱线图法：超过1.5倍四分位距的数据视为异常

3. **数值范围校验**
   - PM2.5: [0, 500] μg/m³
   - PM10: [0, 600] μg/m³
   - AQI: [0, 500]

4. **数据标准化**
   - 使用Min-Max归一化
   - 使用Z-score标准化

### 4.6 预测模型实现

#### 4.6.1 特征工程

```python
def feature_engineering(df):
    """特征工程"""
    # 时间特征
    df['hour'] = df['monitor_time'].dt.hour
    df['day_of_week'] = df['monitor_time'].dt.dayofweek
    df['month'] = df['monitor_time'].dt.month
    df['is_weekend'] = df['day_of_week'].isin([6, 7]).astype(int)
    
    # 滞后特征
    for lag in [1, 6, 12, 24]:
        df[f'pm25_lag_{lag}'] = df.groupby('station_code')['pm25'].shift(lag)
        df[f'pm10_lag_{lag}'] = df.groupby('station_code')['pm10'].shift(lag)
    
    # 同期特征
    df['pm25_same_hour_yesterday'] = df.groupby(
        ['station_code', df['monitor_time'].dt.hour]
    )['pm25'].shift(24)
    
    # 统计特征
    df['pm25_rolling_6h_avg'] = df.groupby('station_code')['pm25'].transform(
        lambda x: x.rolling(window=6, min_periods=1).mean()
    )
    
    return df
```

    y_pred = model.predict(X_test)
    
    mae = mean_absolute_error(y_test, y_pred)
